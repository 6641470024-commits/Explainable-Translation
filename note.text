Frontend: ใช้ Streamlit เพราะสร้าง Web App ได้รวดเร็ว และรองรับ Python ได้เต็มรูปแบบ
AI Engine: ใช้ Google Gemini 2.5 Flash ซึ่งมีความโดดเด่นด้านการประมวลผล Multimodal (อ่านภาพพร้อมข้อความได้แม่นยำ)
Image Processing: ใช้ Pillow (PIL) สำหรับจัดการไฟล์รูปภาพก่อนส่งให้ AI
Environment Management: ใช้ python-dotenv เพื่อความปลอดภัยในการเก็บ API Key



อธิบายส่วนสำคัญ
1. การจัดการความปลอดภัย (Environment Variables)
ทำอะไร: ใช้ load_dotenv() เพื่อดึง API Key จากไฟล์ .env
ทำไมถึงสำคัญ: นี่คือมาตรฐานความปลอดภัยระดับสากล เพื่อไม่ให้ API Key รั่วไหลเมื่อเราแชร์โค้ดลง GitHub หรือส่งต่อให้คนอื่น

2. เทคนิค Prompt Engineering & Multimodal
ทำอะไร: ในฟังก์ชัน analyze_language_content เราสั่งให้ Gemini ทำตัวเป็น "ครูสอนภาษา" ไม่ใช่แค่ตัวแปลภาษา
ทำไมถึงสำคัญ: การระบุโครงสร้าง ### [TRANS] ใน Prompt ช่วยให้เราสามารถใช้คำสั่ง .split() ใน Python เพื่อแยกเนื้อหาที่ AI ตอบกลับมาเข้าสู่ Tabs ต่างๆ ได้อย่างแม่นยำ นี่คือหัวใจที่ทำให้ UI ดูสวยงาม

3. การจัดการสถานะ (Session State)
ทำอะไร: ใช้ st.session_state เก็บ history และ current_view
ทำไมถึงสำคัญ: Streamlit จะรันโค้ดใหม่ทุกครั้งที่กดปุ่ม ถ้าเราไม่ใช้ Session State ข้อมูลที่ AI แปลไว้จะหายไปทุกครั้งที่หน้าจอขยับ การใช้ส่วนนี้ช่วยให้ผู้ใช้ย้อนดูประวัติได้โดยไม่ต้องเสียโควตาเรียก API ใหม่

4. ส่วนการแสดงผล (Data Parsing)
ทำอะไร: การใช้ .split("### [หัวข้อ]") เพื่อดึงเนื้อหาเฉพาะส่วน
ทำไมถึงสำคัญ: เป็นการโชว์กึ๋นในการจัดการข้อความ (String Manipulation) เปลี่ยนข้อความยาวๆ จาก AI ให้กลายเป็น Dashboard ที่อ่านง่าย แบ่งเป็นหมวดหมู่ชัดเจน (คำแปล, ไวยากรณ์, คำศัพท์)

5. รายชื่อภาษา (Symmetrical Logic)
ทำอะไร: การใช้ LANG_OPTIONS ชุดเดียวกันทั้งต้นทางและปลายทาง
ทำไมถึงสำคัญ: ทำให้แอปมีความยืดหยุ่น (Versatility) สูง ไม่ได้จำกัดแค่การแปลเป็นไทย แต่คนชาติอื่นก็สามารถใช้เรียนรู้ภาษาไทยหรือภาษาอื่นๆ ได้เช่นกัน

สรุปจุดเด่นของโค้ดนี้: "เป็นแอปที่ใช้รุ่น Gemini 2.5 Flash เพื่อความรวดเร็ว มีการออกแบบ UI ให้รองรับหลายภาษา (Localization) และมีการจัดการข้อมูลที่ซับซ้อนให้แสดงผลออกมาในรูปแบบที่เข้าใจง่ายครับ"
